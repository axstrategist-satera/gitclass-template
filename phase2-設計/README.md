# Phase 2：設計

企画を「どう作るか」の形に落とすフェーズです。
ここでの提出物は**成果物の種類によって違います**。自分の種類の行を見てください。

## 提出物（共通＋種類別）

**共通（全員必須）**：`設計書.md`（このフォルダに最初から入っているファイルに直接記入。ファイル名は変えない）

**種類別（自分のものだけ）**：

| 成果物の種類 | 設計段階でコミットするもの | 形式の目安 |
|---|---|---|
| プログラミング | 画面遷移図やフローチャート、使う技術のメモ | 画像（PNG）＋ `設計書.md` に説明 |
| 動画 | **絵コンテ**（手描きスキャン or デジタル） | PNG / JPG を `絵コンテ/` フォルダに連番で（例 `01.png`） |
| デザイン | ラフスケッチ・ワイヤーフレーム | PNG / JPG |
| 3D モデリング | 三面図ラフ・寸法メモ・参考写真 | PNG / JPG ＋ `設計書.md` に寸法 |

## 手順

1. このフォルダの `設計書.md` に直接記入する
2. 図・スケッチ類は画像（PNG / JPG）にしてこのフォルダに入れる
   - 紙に描いた場合はスマホで撮影して OK（文字が読める明るさで）
   - **1 枚 10MB 以下**を目安に（スマホ写真はそのままで大体 OK）
3. コミットメッセージ例：`phase2: 絵コンテ 1〜6 カットを追加`

## このフェーズのポイント

- 設計を見た他人が「作るもの」を想像できるか
- Phase 1 の企画とズレていないか（ズレたなら企画書も直してコミット。方針変更は悪いことではありません）
