## 1. タイトル（仮で OK）

「みつかるくん」— 校内の落とし物を減らす掲示アプリ

## 2. 何を作るか（完成イメージを 3 行で）

校内で見つかった落とし物を、職員室に届いた時点で写真つきで一覧掲示できる Web アプリを作る。
生徒は自分のスマホや教室の Chromebook から「いつ・どこで・どんな物」が届いているかを検索でき、心当たりがあれば受け取り予約ができる。
先生側は受け渡しが済んだ物にチェックを付けるだけで一覧から消え、月ごとの落とし物件数のグラフも自動で出る。

## 3. 誰のために作るか（ターゲット）

- どんな人：物をなくした本校の生徒と、落とし物対応をしている職員室の先生
- その人の困りごと・楽しみ：生徒は「なくした物が職員室に届いているか」を確かめる方法が、休み時間に職員室まで行って聞くことしかなく、あきらめてしまう人が多い。先生は同じ問い合わせに何度も答える手間があり、長期間残った落とし物の保管や処分にも困っている。アプリで一覧化されれば、生徒は数十秒で確認でき、先生は問い合わせ対応の時間を減らせる。

## 4. なぜ自分がこれを作りたいのか

自分自身が 1 学期にイヤホンをなくしたとき、職員室に届いていたのに気づいたのが 2 週間後だった経験があるから。掲示板の紙の一覧は更新が遅く、写真もないので自分の物か判断できなかった。情報の授業で習った Web アプリの仕組みを使えば、この「情報が届かない」問題は解決できるはずだと考えた。身近な不便を自分の手で直す経験をしてみたい。

## 6. 完成までの見通し

- Phase 2（設計）でやること：画面遷移図（生徒用一覧・検索・受け取り予約、先生用登録・受け渡し管理）と、落とし物データの項目設計（品名・カテゴリ・発見場所・発見日・写真・状態）を作る。個人情報を載せない掲示ルールも決める。
- Phase 3（制作）でやること：GAS とスプレッドシートで動く最小版を作り、クラスの数人に使ってもらって検索のしやすさを改善する。写真アップロードは後回しにして、まず文字情報だけで動くものを優先する。
- 不安なこと・先生に相談したいこと：落とし物の写真を全校に公開してよいかの判断基準と、職員室のどの先生に登録作業をお願いできるかの運用面。
