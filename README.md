# 探究課題：企画から制作まで（GitClass 課題テンプレート）

このリポジトリは、みんなが取り組む探究課題の「提出フォルダ」のテンプレートです。
**フォーク（自分用のコピーを作ること）** して、自分のフォークに成果物をコミット（保存）していくと、それがそのまま提出になります。

> 詳しい操作手順は、先生から配られる「生徒向けオンボーディング」の資料を見てください。

---

## 大事なルール（最初に必ず読む）

1. **このリポジトリは公開（public）です。** 世界中の誰でも見られます。
   - **実名・学校名・クラス名・顔写真は絶対に書かない／上げない。**
   - 名前を書く場所には、先生に登録した**ハンドル名**だけを使う。
2. 作業は必ず**自分のフォーク**で行う。このテンプレリポジトリ（先生のリポジトリ）には書き込めません。
3. 他の人の作品・画像・音楽を無断で使わない（著作権・ライセンスに注意）。
4. 困ったら消したり作り直したりする前に、まず先生に相談。Git はだいたい元に戻せます。

## 課題の流れ（3 つのフェーズ）

| フェーズ | フォルダ | やること | 主な提出物 |
|---|---|---|---|
| Phase 1 企画 | `phase1-企画/` | 何を・誰のために・なぜ作るかを決める | 企画書（Markdown） |
| Phase 2 設計 | `phase2-設計/` | どう作るかを図や下書きにする | 設計書＋ラフ（絵コンテ／ワイヤーフレーム／スケッチ等） |
| Phase 3 制作 | `phase3-制作/` | 実際に作って仕上げる | 作品データ＋制作ふりかえり |

- 各フェーズには**締切**があります（授業内で案内）。締切までにそのフェーズのフォルダへコミットしてください。
- 一度出して終わりではありません。**フィードバックを受けて何度でも改訂コミットして OK**（むしろ改訂の積み重ねが評価されます）。

## 最初にやること：`meta.yml` で自分の成果物タイプを設定する

フォークしたら、いちばん最初に **リポジトリ直下の `meta.yml`** を開いて、自分が作る作品のタイプ（`code` / `video` / `design` / `model3d`）に書きかえてコミットしてください。書き方はファイル内のコメントに全部書いてあります。
これを設定しておくと、自動フィードバックが自分の作品タイプに合った内容で返ってきます（未設定だと共通の観点だけしかチェックされません）。

## 提出方法（ざっくり）

1. このリポジトリを **Fork** する（最初に 1 回だけ）
2. `meta.yml` を自分の成果物タイプに書きかえてコミットする（最初に 1 回だけ。上の説明を参照）
3. 自分のフォークの該当フェーズのフォルダに、ファイルを追加・編集する
4. **コミットメッセージ**を付けてコミットする
5. GitHub 上で自分のフォークにファイルが表示されていれば提出完了（先生側で自動的に確認されます）

### コミットメッセージの書き方

「いつ・何をしたか」が履歴として残り、これも評価の材料になります。次の形で書いてください。

```
<フェーズ>: <何をしたか>
```

例：

- `phase1: 企画書の初稿を作成`
- `phase1: フィードバックを受けてターゲット像を修正`
- `phase3: BGM の音量バランスを調整`

## ファイルサイズの注意

- **1 ファイル 50MB まで**を目安に。GitHub は 100MB を超えるファイルを受け付けません。
- 動画の完成データなど大きいファイルはコミットせず、各フェーズの README にある方法（書き出し設定の変更・Google Drive リンクなど）で提出します。

## 評価について

コミット履歴（いつ・どれだけ・どう改善したか）と成果物の両方が評価対象です。
評価の観点（ルーブリック）は授業内で配布されます。自動フィードバックが返ってくることがあるので、参考にして改訂しましょう。
